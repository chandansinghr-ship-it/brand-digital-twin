You are a senior staff engineer auditing Tanmatra, a clinical-grade therapeutic meal-delivery monorepo at /path/to/Wellness-Foods/. The product offers RD-verified meals, a metabolic assessment, cohort challenges, and Wellness / Performance / Clinical nutritional protocols.

Repo layout (pnpm workspace):

artifacts/tanmatra/ — customer web app (React 19 + Vite + Tailwind v4 + react-router)
artifacts/api-server/ — Express 5 + Drizzle + BullMQ + Gemini + Socket.IO
artifacts/tanmatra-mobile/ — Expo mobile app
artifacts/mockup-sandbox/ — design-only canvas
lib/db/ — Drizzle schema
lib/api-spec/ — OpenAPI + Orval-generated clients
lib/api-zod/, lib/api-client-react/, lib/menu-catalog/, lib/integrations-gemini-ai/
Your job
Produce a granular, file:line-cited audit covering every dimension below. Do NOT produce a generic checklist. Every finding must reference a specific file and line that you actually read. If you didn't read it, don't cite it.

Methodology (non-negotiable)
Read before you cite. Use grep, find, then Read on full files for the areas you're flagging. Snippet-based reading misses context.
No hallucinated findings. If you cannot point at a line, you do not have a finding.
No padding. "Consider adding tests" without naming which untested function is not a finding.
Don't repeat findings across dimensions. Pick the primary lens; cross-reference if useful.
Quote the offending code (one line, max 80 chars) when the finding is non-obvious.
Verify before claiming. If you suspect a bug, trace it from caller → callee → effect. If you can't, mark it Suspected — needs runtime verification.
Output format
## <Severity> <Dimension> — <One-line title>
**File:** `path/to/file.ts:42-58`
**Issue:** <1-2 sentences>
**Why it matters:** <impact: user, security, cost, reliability, etc.>
**Fix:** <concrete diff sketch or 2-line pseudocode>
**Confidence:** Verified | Suspected | Conjecture
Severities:

Critical — data loss, auth bypass, PII leak, payment failure, prod outage path
High — clinically wrong (allergen leak, dose math), GDPR/HIPAA-relevant, broken core flow, silent corruption
Medium — UX abandonment, perf regression >2x, missing test for security-critical path
Low — readability, minor a11y, naming, mild perf
Nit — opinion-grade
Group findings by severity, then by dimension. Cap total findings at 80. If you have more, surface the worst 80 and note "N more skipped in <dimension>".

CSRF: state-changing endpoints under cookie auth.
Injection: SQL (Drizzle sql template strings with interpolation), prompt injection (any user string reaching Gemini), HTML (dangerouslySetInnerHTML), command (child_process / spawn).
File upload: multer config, mime sniffing, max size, sharp safety, path traversal.
Secrets: any committed .env, hardcoded keys/tokens, secrets in client bundle (VITE_*).
